## 1. 背景与问题定义

AI 生成内容（漫剧、短剧、插画资产等）的生产流程中，同一项目的素材往往来自不同的生成批次，色调互相不统一。生成师需要快速将几百张图片统一成同一个色调（通常参照某张理想的参考图），目前的方案要么是回炉重新生成（慢、贵、抽卡不可控），要么是 PS 批处理（繁琐、无实时反馈）。

**一句话定义**：拖入一张参考图和一批素材图，几百张图实时统一成参考图的色调，批量导出。

**核心竞品差异**：市面上的滤镜/调色工具均为"单张图 + 预设库"模式；本工具的壁垒是"参考图驱动 + 批量处理 + 实时预览"。

## 2. 目标用户


| 用户        | 场景                         | 诉求                 |
| --------- | -------------------------- | ------------------ |
| 生成师（核心用户） | 拿到一批颜色不统一的生成图，需统一成项目世界观的色调 | 快、粗调即可、试错式工作流、素材保密 |
| 半专业创作者    | 插画/摄影/自媒体图片套统一风格           | 简单、好看、零学习成本        |
| 后期人员（次要）  | 将解算出的色调带入 Resolve/PS 等后续流程 | 可导出标准 LUT 文件       |


**明确不服务**：追求精确色彩科学的专业调色师（他们用 DaVinci Resolve）。

---



## 3. 产品原则

1. **速度即人格**：任何操作 <100ms 开始出现视觉反馈（指响应开始时间；过渡动画时长另见 §5.3）；换参考图，所有可见缩略图瞬间跟随变色。
2. **粗调优先**：一根强度滑杆解决 90% 的调整需求，不暴露曲线、色轮等专业概念。
3. **诚实呈现**：不假装一键完美，自动标出迁移效果可疑的图片供用户单独处理（P1，M3）。
4. **本地处理**：全部计算在浏览器端完成，图片不上传服务器（写进 README 第一行）。
5. **色调即资产**：色调保存为标准 .cube 文件——可分享、可署名、可直接进 Resolve/PS，这是社区传播机制的核心。

## 4. 功能需求



### 4.0 优先级口径（本文档唯一标准）

- **P0** = MVP 上线必须（M1–M2 完成），§8 的 ✅ 清单即全部 P0 的汇总
- **P1** = 非上线阻塞的第一批增强，每项单独标注目标里程碑
- **P2** = 远期方向 / 留待社区

如个别表述与本口径冲突，以本节为准。

### 4.1 核心管线（一条管线，四个色调入口，一个出口）

```
色调来源（四选一）──→ 色调变换引擎 ──→ 批量应用 + 实时预览 ──→ 导出
```



#### 入口 ①：参考图驱动（P0，MVP 核心，最高优先级）

- 用户拖入参考图，系统解算出色彩变换（初版算法：Reinhard 统计匹配，Lab 空间）
- **多参考图（v0.6 引入，v0.10 改版，P0，核心功能重点打磨）**：支持 0–20 张参考图，**等权解析为一个色调**统一应用到全部素材；超上限拒绝并提示
- **合成公式 = "平均观感"（v0.10 选优）**：目标 μ/σ 取各参考图 Lab 统计的等权平均。旧 pooled（拼图）语义的方差含"参考图间均值离散"项，参考组差异大时被错误注入为对比度/饱和度膨胀（高光溢出、唇色荧光粉）；平均观感在参考组相似时与 pooled 几乎一致，差异大时给出自然的"平均色调"。真实图对比实验与数学根因见 `m0/output/report.md` 附录
- **强度只有全局一档（v0.10）**：N 张图解析成一个色调，故不设逐张权重滑杆；全局强度 0–100%（默认 70%）
- 实时套用走**逐图匹配**：每张素材用自身统计量对齐合并后的参考统计，GPU 以逐图仿射 uniform 渲染（见 §7，M0 结论）；保存/导出 .cube 时才烘焙为 3D LUT
- 支持随时增删参考图，预览实时跟随；面板内以**大预览卡片 + ‹› 切换 + 下方色板条**展示（v0.11，原型样式）；预览用原图（清晰，BUG-009），统计仍走 256 缩图



#### 入口 ②：内置色调模版（P0，进 MVP）

- MVP 内置 8 个自制通用风格（暖调 / 冷调 / 青橙 / 日系 / 黑白 / 高对比 / 低饱和 / 复古），构建期预烘焙为 33³ LUT，与参考图驱动共用同一渲染管线
- 模版以缩略图网格呈现于右侧调色面板（布局参照交互原型）
- 让没有现成参考图的用户也能立刻体验产品价值
- ⚠️ 不做胶片品牌模拟（Kodak/Fuji 等涉及内容积累与品牌名风险），列为 P2，留待社区贡献



#### 入口 ③：导入色调文件（P0，进 MVP）

- 支持拖入标准 .cube 文件（3D LUT）——本工具保存的色调文件就是标准 .cube，**无自有格式**，来自 Resolve/PS 或他人分享的 .cube 均可直接使用
- **只保留一个导入位（v0.10）**：导入后显示 .cube 名称条（✕ 可删除），再导入即替换当前；导入即生效（激活为当前色调）
- **与参考图共用一个入口模块（v0.11）**：同一个虚线框同时接受图片与 .cube（图片=参考图、.cube=LUT 自动区分），名称条显示在模块内，模块尺寸不变
- 导入本身不进"最近"历史——只有执行导出/下载时色调才存入历史（见 §5.3）
- P2：.3dl、HALD CLUT 等其他通用 LUT 格式



#### 入口 ④：色调保存与导出（P0，进 MVP，差异化功能）

- 将入口 ①/② 的变换保存为**标准 .cube 文件**，可直接用于 Resolve/PS/Premiere，也可重新拖回本工具复用
- 注意语义：.cube 无法承载逐图变换，保存的是**全局变换**（整批统计量 → 参考图），用于跨软件迁移与分享；工具内的"批量统一"效果以逐图匹配为准（见 §7）
- 元数据（色调名称、创建时间、生成参数）写入 .cube 的 `TITLE` 字段与 `#` 注释行
- **强度烘焙**：按当前强度烘焙导出（所见即所得）；v0.11 起去掉"导出 100% 原始强度"勾选与作者署名输入（手写 bug 单⑤⑥，导出区只留两个按钮）；作者署名待"色调卡片"（§5.4）承载
- 预览缩略图、代表色带等富媒体信息不进 LUT 文件，由"色调卡片"PNG 承载（见 §5.4）
- 附加：对任意图片提取"专业色调信息"面板——直方图、Waveform、Vectorscope、RGB Parade、色温/色调估算、黑白场位置、饱和度分布（P2，M3，可独立成页作为引流工具）

| 图片下载（双模式，v0.10） | 原分辨率导出；**保持原文件名与原格式**（png 保留 alpha，webp 保持 webp，jpg 质量默认 0.92 可配置）。**单图视图**：按钮为「下载图片」，直接下载当前查看的这张；**宫格视图**：单击多选素材（选中紫框、未选变暗），按钮按选中数切换（v0.11⑦）——**0 张置灰、1 张「下载图片」直接下载、≥2 张「批量下载图片（n）」**；多张落盘：Chrome/Edge 经 File System Access API 写入用户所选目录，Safari 自动降级为 ZIP（store）打包下载；同名文件追加序号；显示进度（按相对路径重建目录结构随「文件夹导入」降为 P2） | P0     |
| 可疑图标记     | 自动计算迁移前后差异，超阈值的图片角标高亮，可筛选查看                                                                                                                                                      | P2（M3） |
| 单图豁免/单独参考 | 对个别图片排除处理或指定另一张参考图；豁免过后提供"恢复全部应用"一键操作                                                                                                                                            | P1（M3） |
| 会话持久化     | 素材/参考图/导入的 .cube/色调历史/界面状态自动保存（本地 IndexedDB），刷新或重开浏览器完整恢复工作区；上传界面提供"清空工作区"入口（v0.8，BUG-008；v0.10 覆盖色调历史与单导入位）                                                                                                    | P0     |

## 7. 技术方案概要

- **前端**：Web 单页应用（框架自选，React/Svelte 均可），WebGL2 渲染
- **色调解算**：Reinhard 统计匹配（Lab 空间，均值+标准差），Web Worker 中计算参考图与每张素材缩略图的统计量
- **实时预览，双路径（M0 结论修订）**：
  - **参考图驱动 = 逐图匹配路径**：shader 内做 sRGB→Lab→仿射→sRGB，每图仅需 6 个 uniform（逐通道 scale/offset）。M0 实测：共享单 LUT（全局匹配）对靠近批次均值的素材几乎不迁移，只有逐图匹配能把互不统一的批次真正拉齐（`m0/output/report.md`）
  - **LUT 纹理路径**：仅用于"导入 .cube / 内置模版"，3D LUT 上传为 GPU 纹理三线性采样（M0 实测 33³ 保真度 ΔE76 最大 1.6，低于 2.3 JND）
  - 两条路径共用强度滑杆，即 shader 里的 mix 系数
- **内置模版**：构建期预烘焙为 33³ LUT，走 LUT 纹理路径
- **缩略图策略**：导入时生成 ~256px 缩略图用于预览（单图/网格墙两种视图共用），导出时对原图逐张离屏渲染
- **导出管线**：原图解码 → WebGL 离屏渲染套用 LUT → 编码写盘，优先 OffscreenCanvas + Worker 并行流水线，以支撑"500 张 <3 分钟"（逐张 canvas 2D 串行方案达不到）
- **可疑图检测**：迁移前后 Lab 均值偏移量 + 直方图断层检测，超阈值标记（P1，M3）
- **.cube 读写**：3D LUT 序列化为标准 Resolve .cube；元数据按"`TITLE` + `#` 注释行"约定读写，遇不含本工具元数据的第三方 .cube 正常兼容
- ⚠️ **原型效果声明**：交互原型（`docs/ToneSync-standalone.html`）中的调色为 CSS 近似示意（参考图平均色 + mix-blend-mode），**不代表最终效果**；最终效果以 M0 Python 验证的 Reinhard/LUT 结果为准
