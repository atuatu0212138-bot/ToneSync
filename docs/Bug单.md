## BUG-001 图片名称溢出图片框

- **报告**：2026-07-17，M1 demo 手写评审
- **现象**：单图模式下，文件名标签固定显示在画布区左上角；当图片按比例居中留边（letterbox）时，标签落在图片边框外面
- **期望**：名称显示在**图片矩形内左上角**，跟随缩放/平移移动，超长省略号截断，图片移出画布时收边显示（PRD §5.2 v0.4 已固化为需求）
- **根因**：`#img-name` 使用固定 CSS 坐标（画布区 14,14），与图片实际渲染矩形无关
- **修复**：2026-07-17，`web/js/app.js` `syncOverlays()` 改为每帧按 `state._imgRect`（图片当前渲染矩形，含缩放/平移）定位并做画布内收边
- **回归 case**：`web/test/regression.mjs` 断言「图片名在图片框内」——读取名称标签的客户端坐标与图片矩形，验证标签在矩形内

## BUG-002 "＋添加素材"格子边框应为虚线

- **报告**：2026-07-17，M1 demo 第二轮手写评审
- **现象**：底部胶片条末尾的"＋ 添加素材"格子边框画成了实线
- **期望**：虚线边框，与交互原型（`docs/ToneSync-standalone.html` 中 `border:1px dashed`）一致
- **根因**：WebGL 重实现时用整条矩形画边框，未还原原型的 dashed 样式
- **修复**：2026-07-17，`web/js/app.js` 新增 `drawDashedRect()`（短段矩形拼接虚线），＋格子改用虚线绘制
- **回归 case**：`web/test/regression.mjs` 断言「＋格子为虚线边框」——校验绘制路径标记 `window.__plusStyle === 'dashed'`（虚线绘制代码被移除即失败）

## BUG-003 图片移动/缩放时分屏滑杆会错位

- **报告**：2026-07-18，M2a demo 手写评审
- **现象**：单图模式下平移/缩放图片后，分屏对比的分割线"错位"——实测分割线相对图片矩形始终对齐（偏差 <0.01px），真实问题是分割线按**整张图片**的比例定位，图片放大/移出画布后分割线跟着跑出可视区，看起来错位且无法再抓取
- **期望**：分屏分割线始终留在画布内可操作，与画面上的原图/调色后分界一致
- **根因**：split% 以图片全宽为基准（`rect.x + rect.w × split%`），未考虑图片可视区
- **修复**：2026-07-18，`web/js/app.js`——split% 改为以**图片可视区间**（图片矩形与画布的交集）为基准：GL 裁切、DOM 分割线、拖动计算三处统一用 `state._vis`；图片完整可见时行为与旧版完全一致
- **回归 case**：`web/test/regression.mjs` 断言「缩放/移动后分屏线保持可视且对齐」——放大 3× 并拖出画布后，校验分割线 x = 可视区起点 + 可视宽 × split%，且在画布范围内

## BUG-004 平移在画布外松手后，下一次画布点击被吞掉

- **报告**：2026-07-18，回归测试发现（「点墙中图片切回单图」用例失败暴露）
- **现象**：拖动图片平移时若鼠标移出画布/窗口后松手，浏览器不触发 click，`_panMoved` 防误触标志残留为 true，导致下一次画布点击（如点击网格墙格子）被误判为拖拽尾巴而忽略
- **根因**：`_panMoved` 只在下一次 click 中被消费清理，没有兜底
- **修复**：2026-07-18，`web/js/app.js` 平移 mouseup 时 `setTimeout(0)` 兜底清零（click 若正常触发仍先被消费，语义不变）
- **回归 case**：由「点墙中图片切回单图并选中」用例间接覆盖（其执行路径包含"画布外松手 → 点击墙格子"）

## BUG-005 放大后图片溢出胶片条区，分屏线显得"变短/错位"

- **报告**：2026-07-18，用户复测"放大图片还是出现滑杆问题"；截图定位
- **现象**：单图放大后 GL 绘制无区域裁剪，图片画穿底部胶片条区域与顶部预留区；而分屏分割线下沿被限制在胶片条上沿——分割线比可见的原图/调色后分界"短一截"，视觉上即滑杆错位
- **根因**：单图/对比两个绘制 pass 没有 scissor 裁剪
- **修复**：2026-07-18，`web/js/gl.js` 增加 `setClip/clearClip`（scissor，DPR 换算）；`web/js/app.js` 单图内容裁剪到 `[0, 画布高 − 胶片条高]`，与分割线纵向范围一致
- **回归 case**：`web/test/regression.mjs` 断言「单图内容裁剪到胶片条上沿」——校验绘制标记 `window.__clip` 的高度为 `stageH − 110`

## BUG-006 分屏手柄/边缘导航热区挡住滚轮缩放

- **报告**：2026-07-18，同上排查中实测发现（zoom 2.69 → 滚轮无响应）
- **现象**：滚轮落在分屏手柄（默认在画面中央）或左右边缘导航热区上时，wheel 事件被覆盖层截获，画布收不到，缩放"时灵时不灵"
- **根因**：wheel 监听挂在 canvas 上，DOM 覆盖层成为事件目标后不冒泡到 canvas（非祖先）
- **修复**：2026-07-18，`web/js/app.js` wheel 监听移至 `#stage` 容器（所有覆盖层的共同祖先），行为逻辑不变
- **回归 case**：`web/test/regression.mjs` 断言「悬停分屏手柄滚轮可缩放」

## BUG-007 浏览器缓存新旧模块错配 → 渲染循环死亡，画布永久空白

- **报告**：2026-07-18，用户"上传素材后左边区域空的，看不到图片和胶片条"
- **现象**：DOM 界面一切正常，但画布区永久空白。排查：全新加载（dpr=1 与 dpr=2）均无法复现、无报错——问题出在用户浏览器缓存了旧版 ES Module（如无 `setClip` 的旧 gl.js）与新版 app.js 混用，`renderer.setClip is not a function` 在首帧 draw() 抛出，rAF 循环从此不再调度
- **根因**：两层——① `python -m http.server` 不发 Cache-Control，Chrome 启发式缓存可返回旧模块；② `frame()` 无异常兜底，单帧异常即杀死整个渲染循环
- **修复**：2026-07-18——① 新增 `web/serve.py` 开发服务器（所有响应 `Cache-Control: no-store`），README 更新为唯一推荐启动方式；② `frame()` 加 try/catch：单帧异常打 console + toast 提示"强制刷新"，循环继续存活
- **应急处理**：遇到画布空白先强制刷新（⌘⇧R）
- **回归 case**：`web/test/regression.mjs` 断言「渲染循环异常兜底不死亡」——注入一次绘制异常，验证 toast 出现且后续帧继续正常渲染

## BUG-008 刷新后工作区全部丢失，回到主页面

- **报告**：2026-07-18，用户按 BUG-007 提示强制刷新后，上传的素材全部丢失、回到空状态主页面
- **现象**：所有会话状态（素材、参考图、.cube、强度/色调等）只存在内存中，任何刷新/重开都从零开始——"刷新恢复"的指引实际是丢工作的操作
- **期望**：刷新或重开浏览器后，工作区自动恢复到离开时的状态
- **修复**：2026-07-18——新增 `web/js/store.js`（IndexedDB 持久化层）：素材/参考图（含权重）/.cube 原始数据自动落盘，界面状态（强度、色调、视图、页签、选中、分屏）任何变化后 300ms 防抖保存；启动时 `restoreSession()` 完整恢复（不播入场动画/crossfade）；上传界面新增"清空工作区（含已保存的会话）"入口；渲染异常 toast 文案改为"刷新页面即可恢复，工作区已自动保存"。数据仅存本地浏览器 IndexedDB，符合"纯本地处理"承诺
- **回归 case**：`web/test/regression.mjs` 断言「刷新后会话恢复」（重载后素材/参考图/.cube 数量与强度完整还原）与「清空工作区（含持久化）」（清空后重载仍为空）

## BUG-009 添加参考图后大预览模糊

- **报告**：2026-07-20，手写 bug 单①："添加参考图后缩略图太糊，原型文件很清晰"
- **现象**：参考图大预览卡片明显发糊；最初原型（ToneSync-standalone.html）里同尺寸预览是清晰的
- **根因**：v0.10 预览图复用了统计用的 256px 缩图（`decodeToCanvas(blob, THUMB)` 后 `toDataURL`），在 ~264px 宽 × 2x DPR 的卡片上被放大约 2 倍显示
- **修复**：2026-07-20，`web/js/app.js addRefFromBlob()`——预览改用原图 `URL.createObjectURL(blob)`（浏览器按显示尺寸解码，清晰且省内存），256 缩图仅用于 Lab 统计与色板计算；`removeRef` 时 revoke
- **回归 case**：`web/test/regression.mjs` 断言「参考图预览用原图不糊」——校验 `#ref-big.src` 为 `blob:` URL 而非 256 缩图 dataURL

## BUG-010 添加参考图后面板模块变高

- **报告**：2026-07-20，手写 bug 单④："添加参考图后模块大小不要变，现在会变高，这块 UI 严格参照原型文件"
- **现象**：空态虚线框较矮，添加参考图后变成大预览 + 缩略图行，模块整体变高，面板内容跳动
- **根因**：模块高度由内容撑开，空态与有图态内容高度不同
- **修复**：2026-07-20，`web/index.html` + `web/css/style.css`——参考图与导入 .cube 合并为单一模块 `#tone-box`（手写调整②），**固定高度 118px，与原型同高**（预览 96px + 间距 8 + 色板 14，首版 190px 被用户复审"还是太高"后按原型实测值收紧）：空态虚线框、参考图预览（+ 底部色板条，手写调整③）、.cube 名称条均在模块内弹性布局，任何状态组合下模块外高不变
- **回归 case**：`web/test/regression.mjs` 断言「合并模块定高不随内容变化且同原型高」——对比有参考图前后与导入 .cube 前后 `#tone-box` 高度相等且 =118px
